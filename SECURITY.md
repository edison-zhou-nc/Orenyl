# Security Policy

## Supported Versions

Orenyl is currently maintained as a single active release line.

| Version | Supported |
|---|---|
| Latest `main` / newest PyPI release | Yes |
| Older releases | Best effort only |

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

To report a vulnerability privately:

1. Email the maintainer at `security@loremcp.com`.
2. Include the affected version, environment, reproduction steps, and any proof-of-concept details.
3. Share whether the issue is already public or known to downstream users.

If email is not available, use a private GitHub security advisory if enabled for the repository.

## What to Expect

- Initial acknowledgment target: within 3 business days.
- Triage target: within 7 business days.
- Coordinated remediation and disclosure timing will depend on severity, exploitability, and downstream impact.

Please keep details private until a fix or mitigation is available and coordinated disclosure is agreed.

## Scope Notes

Reports are especially helpful for:

- Authentication or authorization bypasses
- Data exposure or cross-tenant access
- Deletion-proof or lineage-integrity violations
- Encryption, secret-handling, or snapshot-restoration issues

## Safe Harbor

We support good-faith security research intended to improve project safety. Avoid actions that would harm users, destroy data, or disrupt shared systems.
