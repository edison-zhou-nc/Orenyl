## Thread Structure

### Post 1

Agent memory that can't be deleted is a liability.

Most memory layers help agents remember.
Very few help you prove where memory came from, delete it safely, and verify it won't resurface.

### Post 2

I built Orenyl: governed memory for MCP agents.

It stores agent memory you can:
- trace to source
- audit
- actually delete

### Post 3

The proof flow is the whole point:

store event -> retrieve derived fact -> trace lineage -> delete source -> verify non-resurfacing

Attach `docs/assets/orenyl_health_demo.gif` once captured.

### Post 4

Vector stores remember. Orenyl governs.

Deleting a row is not the same thing as proving downstream derived memory will not come back.
