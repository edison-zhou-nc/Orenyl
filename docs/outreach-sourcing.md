# Outreach Sourcing

## Goal

Build a list of 30-50 relevant people who are likely to care about governed memory, not generic memory infrastructure.

## Priority Source Pools

### MCP

- maintainers of MCP servers
- contributors to MCP server repos
- people discussing memory, tools, or governance in MCP contexts

### GitHub

- authors and contributors in memory-related repos
- maintainers of agent-tooling infrastructure
- people active in issues around memory persistence, auditability, or deletion

### Reddit

- posters and commenters discussing agent memory problems
- relevant communities around MCP, local agent stacks, and builder workflows

### X

- builders posting about MCP, agent memory, retrieval problems, or auditability pain
- founders and engineers showing high-trust AI workflows in public

### Health

- teams building health-adjacent copilots or agents
- technical builders where the hero health-style demo will resonate as a sensitive-memory proof

## Segmentation

- warm: people already adjacent to the repo, ecosystem, or your network
- plausible direct: builders clearly working on adjacent problems
- community-only: good fit for public posting, not necessarily cold DM

## What To Look For

- visible concern about agent memory quality
- concern about deletion, auditability, or compliance boundaries
- real MCP or agent implementation work
- enough technical context that the quickstart is a realistic ask

## What To Avoid

- generic AI influencers with no builder context
- procurement-led enterprise contacts at this stage
- compliance-only contacts expecting certifications first
