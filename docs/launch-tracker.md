# Launch Tracker

## Purpose

Track awareness, engagement, activation, and follow-up during the first launch wave.

## Core Metrics

- stars
- forks
- page views
- install attempts (manual signal)
- deletion verification completions (manual signal)
- response rate
- follow-up status

## Suggested Table

| Date | Channel | Activity | Stars | Forks | Page Views | Install Attempts | Deletion Verification | Response Rate | Notes | Follow-Up |
|------|---------|----------|-------|-------|------------|------------------|----------------------|---------------|-------|-----------|
| 2026-04-10 | setup | tracker created | 0 | 0 | 0 | 0 | 0 | 0% | baseline | none |

## Definitions

- `page views`: GitHub traffic views or equivalent visibility signal
- `install attempts`: manually observed evidence someone tried the quickstart through replies, issues, or direct outreach
- `deletion verification`: manually observed evidence someone reached the core proof step
- `response rate`: replies divided by outreach sent
- `follow-up`: whether a next action is scheduled

## What Good Looks Like

- awareness grows without confusing the audience
- install attempts turn into deletion verification completions
- replies reveal whether the product story is resonating
