# Governance

## Decision Making

Architectural decisions are documented as ADRs in [docs/ADR.md](docs/ADR.md). Significant changes should be discussed in a GitHub Issue or Discussion before implementation.

## Maintainers

The project is maintained by the Lore Maintainers. The current primary maintainer is the repository owner, [Edison Zhou](https://github.com/edison-zhou-nc), who has final decision authority on releases and architectural direction.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). Be respectful, constructive, and inclusive.

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md) for development setup, testing requirements, and PR guidelines.

## Getting Started as a Contributor

1. Pick an issue labeled `good first issue`
2. Comment on the issue to claim it
3. Fork, branch, implement, test, PR
4. PRs require passing CI and one review
