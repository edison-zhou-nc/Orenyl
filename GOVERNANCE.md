# Governance

## Decision Making

Architectural decisions are documented as ADRs in [docs/ADR.md](docs/ADR.md). Significant changes should be discussed in a GitHub Issue or Discussion before implementation.

## Maintainers

The project is currently maintained by the repository owner, [Edison Zhou](https://github.com/edison-zhou-nc), who has final decision authority on releases and architectural direction. Additional maintainers may be added by the current maintainer as the project grows.

## Code of Conduct

This project follows [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Be respectful,
constructive, and inclusive.

## Security Reporting

Sensitive security issues should be reported privately per
[SECURITY.md](SECURITY.md), not filed as public GitHub issues.

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md) for development setup, testing requirements, and PR guidelines.

## Getting Started as a Contributor

1. Pick an issue labeled `good first issue`
2. Comment on the issue to claim it
3. Fork, branch, implement, test, PR
4. PRs require passing CI and one review

## Releases

See [docs/RELEASE_PROCESS.md](docs/RELEASE_PROCESS.md) for the release workflow.
